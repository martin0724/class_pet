# 课堂宠物养成游戏开发计划

基于 [game-design-proposal.md](file:///workspace/game-design-proposal.md) 策划文档，制定以下开发步骤：

---

## 开发阶段总览

| 阶段 | 名称 | 周期 | 主要产出 |
|------|------|------|----------|
| Phase 1 | 基础环境搭建 | 1天 | 项目框架、数据库、基础配置 |
| Phase 2 | 后端核心功能 | 3天 | API接口、数据模型、业务逻辑 |

## Phase 1：基础环境搭建（第1天）

### 1.1 项目初始化
- 初始化 Python Flask 项目
- 创建项目目录结构
- 安装依赖包（Flask、SQLAlchemy、sqlite3等）

### 1.2 数据库设计与初始化
- 创建 SQLite 数据库文件
- 定义数据库模型（学生表、宠物表、互动机会表、日志表）
- 生成数据库迁移脚本
- 执行数据库初始化

### 1.3 基础配置
- 配置 Flask 应用参数
- 设置静态文件路径
- 配置日志记录

### 1.4 产出物
- `app.py` - Flask 应用入口
- `database.py` - 数据库模型定义
- `requirements.txt` - 依赖列表
- `config.py` - 配置文件

## Phase 2：后端核心功能开发（第2-4天）

### 2.1 学生管理 API
- `POST /api/students` - 创建学生
- `GET /api/students` - 获取学生列表
- `GET /api/students/{id}` - 获取单个学生信息
- `PUT /api/students/{id}` - 更新学生信息
- `DELETE /api/students/{id}` - 删除学生

### 2.2 宠物管理 API
- `POST /api/pets` - 创建宠物（领养）
- `GET /api/pets` - 获取宠物列表
- `GET /api/pets/{id}` - 获取宠物详情
- `PUT /api/pets/{id}` - 更新宠物信息
- `DELETE /api/pets/{id}` - 删除宠物

### 2.3 互动机会 API
- `POST /api/points/add` - 增加互动机会
- `GET /api/points/{student_id}` - 获取互动机会
- `POST /api/points/use` - 使用互动机会

### 2.4 宠物互动 API
- `POST /api/interactions/feed` - 喂养宠物
- `POST /api/interactions/play` - 与宠物玩游戏
- `POST /api/interactions/pet` - 抚摸宠物
- `POST /api/interactions/clean` - 清理房间
- `POST /api/interactions/walk` - 外出散步

### 2.5 比赛系统 API
- `POST /api/games/create` - 创建比赛
- `GET /api/games` - 获取比赛列表
- `POST /api/games/join` - 报名比赛
- `POST /api/games/start` - 开始比赛

### 2.6 被动衰减任务
- 实现每日属性衰减逻辑
- 创建定时任务或登录时计算衰减

### 2.7 产出物
- `routes/` - API路由模块
- `models/` - 数据模型
- `services/` - 业务逻辑层
- API接口文档

## 关键技术实现要点

### 1. 滑动交互实现
- 使用 CSS `overflow-x: scroll` + `scroll-snap-type` 实现平滑滑动
- 或使用 JavaScript 实现自定义滑动逻辑

### 2. 宠物动画
- 使用 CSS 动画实现待机呼吸、眨眼效果
- 使用 Lottie 动画库实现复杂互动动画

### 3. 数据持久化
- 使用 SQLite 本地存储
- 每次操作后自动保存

### 4. 属性衰减计算
- 首次登录时计算上次登录到现在的衰减
- 使用日期差计算衰减天数

## 注意事项

1. **儿童友好设计**：界面简洁、按钮大、颜色鲜艳
2. **数据安全**：本地存储，无需登录，保护学生隐私
3. **性能要求**：页面加载时间 < 2秒，动画流畅
4. **兼容性**：支持主流浏览器（Chrome、Firefox、Safari）
5. **可扩展性**：代码结构清晰，便于后续功能扩展
